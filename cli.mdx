# Laddr CLI Reference

Complete reference for all Laddr Command Line Interface (CLI) commands, options, and usage examples.

## Overview

The Laddr CLI is a production-grade, modular command-line interface for building and managing multi-agent AI systems. It provides:

- **Project scaffolding** - Initialize complete project structures
- **Agent management** - Add, configure, and run agents
- **Docker orchestration** - Manage containerized environments
- **Observability** - Built-in tracing, metrics, and diagnostics
- **Development tools** - Hot reload, logging, and debugging

---

## Installation

<CodeBlock code={`# Install Laddr
pip install laddr

# Verify installation
laddr --version

# Get help
laddr --help`} language="bash" />

## Global Options

Available for all commands:

- `-h, --help` - Show help message and exit
- `--version` - Show version and exit
- `--debug` - Enable debug mode with full stack traces

## Project Setup

### init

Initialize a new Laddr project with complete scaffolding.

**Synopsis:**
<CodeBlock code={`laddr init [PROJECT_NAME] [OPTIONS]`} language="bash" />

**Description:**
Creates a new Laddr project with configuration files, Docker setup, default agents, example tools, and documentation.

**Options:**
- `PROJECT_NAME` - Name of the project (interactive if omitted)
- `--path` - Path to create project in (default: current directory)

**Examples:**
<CodeBlock code={`# Interactive mode
laddr init

# Specify project name
laddr init myproject

# Create in specific directory
laddr init myproject --path /path/to/projects`} language="bash" />

---

### add

Add new agents or tools to an existing project.

#### add agent

Add a new agent to the project.

**Synopsis:**
<CodeBlock code={`laddr add agent AGENT_NAME [OPTIONS]`} language="bash" />

**Options:**
- `--role` - Agent role
- `--goal` - Agent goal
- `--backstory` - Agent backstory (optional)
- `--llm-provider` - LLM provider (default: gemini)
- `--llm-model` - LLM model (default: gemini-2.5-flash)

**Examples:**
<CodeBlock code={`# Interactive mode
laddr add agent summarizer

# Fully specified
laddr add agent summarizer \\
  --role "Data Summarizer" \\
  --goal "Analyze and summarize data"`} language="bash" />

#### add tool

Add a new tool to an agent.

**Synopsis:**
<CodeBlock code={`laddr add tool TOOL_NAME [OPTIONS]`} language="bash" />

**Options:**
- `--agent` - Agent to attach tool to
- `--description` - Tool description

**Examples:**
<CodeBlock code={`# Interactive mode
laddr add tool calculator

# Specify agent
laddr add tool scraper --agent researcher`} language="bash" />

## Running & Execution

### run

Run environments, agents, or pipelines.

#### run dev

Run the Laddr development environment with all services.

**Synopsis:**
<CodeBlock code={`laddr run dev [OPTIONS]`} language="bash" />

**Description:**
Starts the complete Laddr stack: PostgreSQL, Redis, MinIO, API Server, Dashboard (http://localhost:5173), and Agent Workers.

**Options:**
- `--build` - Force rebuild Docker images
- `--detach, -d` - Run in detached mode (background)

**Examples:**
<CodeBlock code={`# Start with live logs
laddr run dev

# Start and run in background
laddr run dev --detach

# Force rebuild images
laddr run dev --build`} language="bash" />

#### run agent

Run a single agent locally using the AgentRunner.

**Synopsis:**
<CodeBlock code={`laddr run agent AGENT_NAME [OPTIONS]`} language="bash" />

**Options:**
- `--inputs` - JSON dictionary of inputs (default: empty object)

**Examples:**
<CodeBlock code={`# Run with query
laddr run agent researcher --inputs '{"query":"What is Laddr?"}'

# Run with no inputs
laddr run agent coordinator`} language="bash" />

#### run pipeline

Run a multi-stage pipeline defined in YAML.

**Synopsis:**
<CodeBlock code={`laddr run pipeline PIPELINE_FILE`} language="bash" />

**Examples:**
<CodeBlock code={`# Run pipeline
laddr run pipeline workflow.yml`} language="bash" />

#### run replay

Replay a previous job by job ID.

**Synopsis:**
<CodeBlock code={`laddr run replay JOB_ID [OPTIONS]`} language="bash" />

**Options:**
- `--reexecute` - Re-run the job instead of returning stored result

**Examples:**
<CodeBlock code={`# Retrieve stored result
laddr run replay abc123-456-def

# Re-execute the job
laddr run replay abc123-456-def --reexecute`} language="bash" />

---

### prompt

Run and manage prompt executions.

#### prompt run

Execute a prompt with an agent.

**Synopsis:**
<CodeBlock code={`laddr prompt run PROMPT_NAME [OPTIONS]`} language="bash" />

**Options:**
- `--input, -i` - Input as KEY=VALUE (repeatable)
- `--json, -j` - Input as JSON string or @file.json
- `--wait/--no-wait` - Wait for completion (default: true)
- `--timeout` - Timeout in seconds (default: 60)

**Examples:**
<CodeBlock code={`# Simple input
laddr prompt run researcher --input query="What is Laddr?"

# JSON input
laddr prompt run processor --json '{"files": ["data.csv"]}'

# From file
laddr prompt run researcher --json @research-query.json`} language="bash" />

#### prompt list

List recent prompt executions.

**Synopsis:**
<CodeBlock code={`laddr prompt list [OPTIONS]`} language="bash" />

**Options:**
- `--limit` - Maximum number to show (default: 20)

**Examples:**
<CodeBlock code={`# Show last 20 prompts
laddr prompt list

# Show last 100 prompts
laddr prompt list --limit 100`} language="bash" />

---

### run-local

Run an agent locally using in-memory components.

**Synopsis:**
<CodeBlock code={`laddr run-local AGENT [OPTIONS]`} language="bash" />

**Description:**
Executes an agent in fully local mode without external dependencies (no Redis, PostgreSQL, or Docker required). Fast startup for development and testing.

**Options:**
- `--input` - JSON input to the agent (default: empty object)

**Examples:**
<CodeBlock code={`# Run with no input
laddr run-local researcher

# Run with input
laddr run-local researcher --input '{"query":"AI trends"}'`} language="bash" />

## Monitoring & Debugging

### logs

View logs for an agent worker.

**Synopsis:**
<CodeBlock code={`laddr logs AGENT_NAME [OPTIONS]`} language="bash" />

**Options:**
- `--follow, -f` - Follow log output (live tail)
- `--tail` - Number of lines to show from end

**Examples:**
<CodeBlock code={`# View all logs
laddr logs researcher

# Follow logs
laddr logs researcher --follow

# Show last 100 lines
laddr logs researcher --tail 100`} language="bash" />

---

### ps

Show status of all services and workers.

**Synopsis:**
<CodeBlock code={`laddr ps [OPTIONS]`} language="bash" />

**Options:**
- `--format, -f` - Output format (table or json)

**Examples:**
<CodeBlock code={`# Show status (table format)
laddr ps

# Show status (JSON format)
laddr ps --format json`} language="bash" />

---

### check

Run diagnostic checks for the Laddr runtime.

**Synopsis:**
<CodeBlock code={`laddr check [OPTIONS]`} language="bash" />

**Description:**
Executes the DiagnosticAgent to validate system configuration and health including database connectivity, Redis connection, MinIO access, LLM API connectivity, agent configuration, and environment variables.

**Options:**
- `--output` - Path to write JSON report (default: diagnostic_report.json)

**Examples:**
<CodeBlock code={`# Run diagnostics
laddr check

# Save to custom path
laddr check --output reports/diagnostic.json`} language="bash" />

---

### infra

Show infrastructure configuration and status.

**Synopsis:**
<CodeBlock code={`laddr infra [OPTIONS]`} language="bash" />

**Description:**
Displays the current Laddr infrastructure configuration including queue backend, database, LLM backend and model, storage, cache configuration, and observability settings.

**Options:**
- `--show-config/--no-show-config` - Show configuration values (default: true)

**Examples:**
<CodeBlock code={`# Show full configuration
laddr infra

# Hide sensitive values
laddr infra --no-show-config`} language="bash" />

## Service Management

### scale

Scale an agent worker to N replicas.

**Synopsis:**
<CodeBlock code={`laddr scale AGENT_NAME REPLICAS`} language="bash" />

**Description:**
Horizontally scales an agent worker to the specified number of replicas. Useful for handling high load and parallel processing.

**Examples:**
<CodeBlock code={`# Scale to 3 replicas
laddr scale researcher 3

# Scale to 1 replica
laddr scale researcher 1

# Stop all replicas
laddr scale researcher 0`} language="bash" />

---

### stop

Stop all Laddr services.

**Synopsis:**
<CodeBlock code={`laddr stop [OPTIONS]`} language="bash" />

**Options:**
- `--volumes, -v` - Remove named volumes (warning: data loss)

**Examples:**
<CodeBlock code={`# Stop services (preserve data)
laddr stop

# Stop services and remove volumes
laddr stop --volumes`} language="bash" />

laddr init email-campaign

# Add custom agent
laddr add agent emailer \\
